# Templates

Copy-paste-ready files for an Agent OS workspace. Each template has realistic placeholders — fill them in, don't leave `{{...}}` shipped.

| File | Purpose |
|---|---|
| [AGENTS.md.template](AGENTS.md.template) | Repo-specific instructions any coding agent will read |
| [SOUL.md.template](SOUL.md.template) | Identity, voice, mission, refusal style |
| [MEMORY.md.template](MEMORY.md.template) | Durable memory index |
| [USER.md.template](USER.md.template) | User profile + preferences |
| [TOOLS.md.template](TOOLS.md.template) | Tool inventory with risk levels |
| [SKILL.md.template](SKILL.md.template) | Skill spec with progressive disclosure |
| [MCP_SERVER.md.template](MCP_SERVER.md.template) | Documenting an MCP integration |
| [EVAL_PLAN.md.template](EVAL_PLAN.md.template) | What you'll evaluate and how |
| [GUARDRAILS.md.template](GUARDRAILS.md.template) | Policy, refusals, escalation |
| [HUMAN_APPROVAL_POLICY.md.template](HUMAN_APPROVAL_POLICY.md.template) | Who approves what, when |
| [AGENT_RELEASE_CHECKLIST.md](AGENT_RELEASE_CHECKLIST.md) | Pre-ship gate |

## Conventions

- Replace `{{placeholders}}`
- Keep each file short — under ~150 lines is a good ceiling
- One responsibility per file
- Date entries with `(YYYY-MM-DD)` so they can be expired
